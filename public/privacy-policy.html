Dockmarks — Privacy Policy

Last updated: April 2026

What data does Dockmarks collect?

Dockmarks does not collect, transmit, or share any personal data. It does not use analytics, tracking pixels, or third-party services.

Data stored locally

The following data is stored on your device only, using Chrome's built-in storage APIs:

  • Bookmark JSON URL — the URL you configure to load bookmarks from.
  • Cached bookmarks — a local copy of the JSON data fetched from the URL above.
  • Theme preference — your selected visual theme (Neutral or Terminal).
  • Recently used bookmarks — the IDs of the last 10 bookmarks you clicked.
  • Sync metadata — last sync timestamp and content hash for change detection.

Network requests

Dockmarks makes a single type of network request: an HTTP GET to the bookmark JSON URL you configure. No data is sent to any other server. No cookies or authentication tokens are transmitted.

Permissions

  • sidePanel — required to display the bookmark panel in Chrome's side panel.
  • storage — required to persist your settings and cached bookmarks across sessions.
  • alarms — required to schedule periodic background sync (every 60 minutes).

No host permissions are requested. The extension cannot read or modify any web page content.

Third parties

Dockmarks does not share data with any third party. There are no ads, analytics, or external SDKs.

Contact

If you have questions about this privacy policy, open an issue at the project's GitHub repository.
